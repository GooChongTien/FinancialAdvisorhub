## ğŸ“ Documentation Guidelines

### For Contributors

If you're updating or creating documentation:

1. **Be Clear**: Use simple, direct language
2. **Be Specific**: Include exact steps and screenshots
3. **Be Consistent**: Follow existing formatting
4. **Be Current**: Update version numbers and dates
5. **Be Helpful**: Consider the reader's perspective

### Markdown Standards

- Use ATX-style headers (`#`, `##`, `###`)
- Include table of contents for long documents
- Use code blocks for technical examples
- Add screenshots where helpful (keep <500KB)
- Link to related documentation

### Version Control

- Update "Last Updated" date when editing
- Increment version number for major changes
- Document changes in version history
- Archive old versions when superseded
